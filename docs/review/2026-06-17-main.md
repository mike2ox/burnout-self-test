# 코드 리뷰 피드백 — main (2026-06-17)

## 원본 피드백
지금보니까 email로 sign in 하는 링크가 오네? 뭐 이렇게도 괜찮긴한데, OTP나 email로 코드를 받아서 로그인하는게 덜 번거로울거 같아.
그리고 번아웃 지표 트래킹이랑, 나만의 번아웃 탈출방법도 auth를 했으니까 계정별로 묶어서 할 수 있지 않나?

## 분류 결과

### 필수 수정 (보안 / 데이터 격리)
- **계정별 데이터 분리 누락**: history/methods 테이블에 user_id가 없어 모든 인증 사용자가 데이터를 공유·삭제 가능한 상태
  - ✅ 적용 (DB): `user_id uuid references auth.users(id) default auth.uid() not null` 컬럼 추가 필요 (사용자 직접 SQL 실행)
  - ✅ 적용 (DB): RLS를 `auth.role() = 'authenticated'` → `auth.uid() = user_id` 기반으로 교체 필요 (사용자 직접 SQL 실행)
  - ✅ 적용 (코드): `currentUserId` 상태 추가, `onSignedIn(user)`에서 `user.id` 보관

### 권장 개선 (UX)
- **Magic link → 6자리 OTP 코드 방식 전환**: `emailRedirectTo` 옵션이 있으면 Supabase가 링크 이메일을 발송
  - ✅ 적용 (코드): `signInWithOtp`에서 `emailRedirectTo` 제거 (commit: fix: OTP 코드 전용 로그인 전환)
  - ⏳ 대시보드 작업 필요: Authentication → Email Templates → "Magic Link" 템플릿에서 `{{ .ConfirmationURL }}` → `{{ .Token }}` 으로 교체

### 선택적 제안
- **OTP 만료 안내 및 재전송 버튼**: 현재 오류 메시지가 "코드가 올바르지 않습니다"로 고정 — 미적용 (범위 외)
- **clearAllHistory 명시적 user_id 필터**: RLS uid 기반 전환 완료 후 자동 처리되므로 미적용
